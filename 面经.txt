shared_ptr：
每一个 shared_ptr 内部都维护一个引用计数（use count）：
每拷贝一个 shared_ptr，引用计数 +1；
每销毁一个 shared_ptr，引用计数 -1；
当引用计数变为 0 的那一刻，shared_ptr 管理的对象会自动被 delete。

虚函数表（vtable）机制（简述）：
每个含有虚函数的类，编译器会创建一个“虚函数表”（vtable）
每个对象中包含一个“指向 vtable 的指针”
调用虚函数时，程序会根据对象的真实类型查 vtable，调用正确的函数地址

析构函数是否必须是virtual 的：
如果你打算通过基类指针或引用来删除派生类对象，那么基类析构函数必须是 virtual 的，否则会导致资源泄露。
为什么会这样？（背后的原理）
**虚函数机制（vtable）**支持运行时根据对象实际类型做“动态绑定”。
如果析构函数是虚函数，delete 会通过 vtable 找到真实类型的析构函数（即先析构 Derived，再析构 Base）。
如果不是虚的，只会静态绑定基类的析构函数。

不可以：C++ 构造函数不能是虚函数（virtual）。
虚函数依赖 vtable / vptr（虚函数表机制）。而对象在进入构造函数时，对象的“动态类型”还没构造完成，vptr 也会在构造过程中逐步设置成当前正在构造的那一层类型。

在 C++ 中，不能给整个类加 static 修饰符，不像 Java。
不打算创建对象实例，只希望通过类名调用成员函数。
所有成员函数和变量 必须是 static，否则就无法在没有对象的情况下调用。

函数前面后面加const的区别：
const int func();	返回值是 const int（返回后不能修改）	const int getValue();
int func() const;	成员函数是 const 函数，不能修改成员变量	int getId() const;
